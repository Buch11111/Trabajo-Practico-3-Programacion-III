Fundamentos · Stream<T> 02 / 21
¿Qué es un Stream?
🚫 No almacena datos
Un Stream es una vista sobre una fuente (Collection, array, I/O). No tiene su propio almacenamiento — no modifica la fuente original.
💤 Evaluación lazy
Las operaciones intermedias no se ejecutan hasta que una operación terminal las desencadena. Evita trabajo innecesario.
1️⃣ Consumible una vez
Una vez que el Stream es consumido por una operación terminal, no puede reutilizarse. Llamarlo dos veces lanza IllegalStateException.

Operaciones Intermedias 03 / 21
Operaciones intermedias — filtrar, transformar, ordenar
filter(Predicate) Stateless
Conserva elementos que satisfacen el predicado.
.filter(c -> c.getRating() >= 4)
map(Function) Stateless
Transforma cada elemento a otro tipo.
.map(this::toDTO)
sorted(Comparator) Stateful
Ordena el stream. Requiere ver todos los elementos.
.sorted(Comparator.comparingInt(Cancion::getReproducciones).reversed())
limit(long) Short-circuit
Trunca el stream al número indicado. Muy eficiente.
.limit(10) // top10MasReproducidas()
distinct() Stateful
Elimina duplicados usando equals(). Usa un Set interno.
.map(Cancion::getGenero).distinct()
flatMap(Function) Stateless
Aplana streams anidados. Esencial para colecciones de colecciones.
.flatMap(album -> album.getCanciones().stream())

Búsqueda Binaria + Comparator 13 / 21
Búsqueda binaria con Streams + Comparator
BusquedaService.java 
1public Optional<Cancion> busquedaBinariaPorTitulo(
2        List<Cancion> canciones, String titulo) {
3
4    // O(n log n): ordenar antes de buscar
5    List<Cancion> ordenadas = canciones.stream()
6        .sorted(Comparator.comparing(
7            Cancion::getTitulo,
8            String.CASE_INSENSITIVE_ORDER))
9        .toList();
10
11    int left = 0, right = ordenadas.size() - 1;
12
13    while (left <= right) {
14        int mid = left + (right - left) / 2;
15        int cmp = ordenadas.get(mid).getTitulo()
16                              .compareToIgnoreCase(titulo);
17        if (cmp == 0) return Optional.of(ordenadas.get(mid));
18        else if (cmp < 0) left = mid + 1;
19        else right = mid - 1;
20    }
21    return Optional.empty();
22}
Complejidad
O(n log n)
sort + O(log n) búsqueda
Retorna
Optional<T>
sin NPE, caller decide
Combinación de APIs
Stream.sorted() — genera lista inmutable ordenada
Comparator.comparing() — key extractor + comparador específico
String.CASE_INSENSITIVE_ORDER — Comparator listo para strings
Optional.of() / empty() — retorno semántico sin null
⚠️ Consideración de diseño
Si el catálogo ya está ordenado y es estable, la lista puede ordenarse una vez en el repositorio y reutilizarse. El sort dentro del método es correcto para listas volátiles.

1@Repository
2public class CancionRepository {
3
4    // Thread-safe sin synchronized explícito
5    private final ConcurrentHashMap<UUID, Cancion> canciones
6        = new ConcurrentHashMap<>();
7
8    public List<Cancion> findAll() {
9        // values() + ArrayList = snapshot inmutable del estado
10        return new ArrayList<>(canciones.values());
11    }
12
13    public Optional<Cancion> findById(UUID id) {
14        // Map.get() retorna null si no existe
15        return Optional.ofNullable(canciones.get(id));
16    }
17
18    public void saveAll(List<Cancion> lista) {
19        // forEach funcional sobre la lista
20        lista.forEach(c -> canciones.put(c.getId(), c));
21    }
22}
Puntos clave del diseño
ofNullable — el puente entre Map y Optional
Map.get() retorna null cuando la clave no existe. ofNullable convierte ese null en un Optional semántico sin if-null checks.
new ArrayList<>(values()) — snapshot defensivo
Los Streams operan sobre la copia. Mutaciones concurrentes al Map durante el pipeline no corrompen el stream en proceso.
forEach en saveAll
Equivalente funcional al for-each. ConcurrentHashMap.put() es thread-safe: safe bajo múltiples threads sin locks externos.

@RestControllerAdvice 
@RestControllerAdvice
public class GlobalExceptionHandler {

  @ExceptionHandler(CancionNotFoundException.class)
  public ProblemDetail handleCancionNotFound(
        CancionNotFoundException ex) {
    ProblemDetail detail = ProblemDetail
        .forStatusAndDetail(HttpStatus.NOT_FOUND, ex.getMessage());
    detail.setTitle("Canción no encontrada");
    detail.setProperty("timestamp", Instant.now());
    return detail;
  }
  // + handlers para Artista, Album, Productora,
  //   IllegalArgumentException, Exception (fallback)
}
¿Por qué centralizado?
El service lanza una excepción de dominio (CancionNotFoundException) sin saber HTTP. El advice traduce a status code una sola vez para toda la API — sin duplicar try/catch en cada controller.

API Cuándo usarla Ejemplo en el proyecto Evitar
Stream<T> Transformar / filtrar colecciones declarativamente top10MasReproducidas() streams infinitos sin límite
Optional<T> Retorno que puede estar ausente (repositorios, búsqueda) findById(UUID) como campo de clase

Collectors Agregar, agrupar, resumir datos promedioRatingPorGenero() collect en streams pequeños
IntStream Operaciones numéricas masivas (sum, avg, max) mapToInt(Cancion::getReproducciones) map a Integer (boxing)
@FunctionalInterface Contratos con una única función (Strategy, Command) EstrategiaRecomendacion más de un método abstracto
❌ Antipatrones comunes
· optional.get() sin check
· Stream reutilizado
· sorted() antes de filter()
· peek() para lógica de negocio
✅ Buenas prácticas
· filter() antes de sorted()
· mapToInt sobre int primitivos
· toList() vs collect()
· reduce con identidad explícita
⚡ Rendimiento
· Stateless ops primero, stateful al final
· Short-circuit con findFirst
· limit() es lazy
· Parallel streams: solo para datos grandes
